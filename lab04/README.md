Паттерн "Плагин"
Задача:
Разработать приложение для ведения личных финансов, которое должно обладать следующими возможностями:
Добавление, редактирование и удаление транзакций
Фильтрация по месяцам и категориям
Отслеживание прогресса накоплений
Управление регулярными платежами

С паттерном:
IFinancePlugin (интерфейс плагина)
Базовый интерфейс, который должны реализовывать все подключаемые модули. Он определяет три метода:
String getName() – возвращает название плагина (отображается на кнопке в интерфейсе);
String getDescription() – краткое описание функциональности (отображается в подсказке);
void execute()– основной метод, запускающий логику плагина. 

SavingsGoalPlugin (финансовая цель)
Внутри содержит:
создание таблицы savings_goals при первом запуске
диалоговое окно для создания/редактирования/удаления целей
расчёт прогресса на основе текущего баланса транзакций
отображение списка целей с процентами выполнения.

PaymentCalendarPlugin (платёжный календарь)
Внутри содержит:
создание таблицы scheduled_payments для хранения регулярных платежей
календарь с подсветкой дней, на которые запланированы платежи
добавление/редактирование/удаление событий (платежей)
автоматическое создание транзакций из событий за выбранный месяц
отображение списка всех регулярных платежей.

FinanceApp (главное приложение)
Основной класс приложения выполняет следующие функции, связанные с плагинами:
В методе loadPlugins() приложение сканирует папку plugins в рабочем каталоге, находит все JAR-файлы. Для каждого JAR создаётся отдельный загрузчик классов URLClassLoader. С помощью ServiceLoader (стандартный механизм Java для поиска сервисов) находятся все классы, реализующие IFinancePlugin. Найденные экземпляры сохраняются в список plugins.
Создание кнопок. Для каждого загруженного плагина создаётся кнопка в. Название кнопки берётся из plugin.getName(), всплывающая подсказка – из plugin.getDescription().
Вызов плагина. При нажатии на кнопку приложение определяет текущий выбранный месяц в фильтре, преобразует его в начальную и конечную даты, вызывает метод execute() плагина, передавая соединение с БД, ссылку на главное окно и даты периода. После завершения работы плагина приложение автоматически обновляет список транзакций.

Без паттерна:
Вся логика SavingsGoalPlugin и PaymentCalendarPlugin перенесена внутрь FinanceApp в виде private-методов.
Кнопки вызова функционала добавлены прямо в панель фильтров (рядом с фильтрами).
Удален механизм подключения плагинов.
Остальные классы (DatabaseManager, TransactionDialog) остаются без изменений.

Сравнение:
Применение плагинов пощзволило лучше структурировать код, а также сократить связность кода, что делает его более гибким для изменений.
